# Experiment example workflows

This page highlights the main notebook entry points for the `Experiment` workflow.
Use these notebooks after completing the [Quickstart](../getting-started/quickstart.md).

## Recommended starting points

- [Basic usage](../../examples/experiment/0_basic_usage.ipynb): Create an experiment session and run the core hardware-backed workflow.
- [Spectroscopy](../../examples/experiment/1_spectroscopy.ipynb): Find resonator and qubit frequencies from standard scans.
- [Frequency calibration](../../examples/experiment/2_frequency_calibration.ipynb): Refine drive and readout frequencies.
- [Pulse calibration](../../examples/experiment/3_pulse_calibration.ipynb): Tune control pulses used in gate operations.
- [T1 and T2 experiments](../../examples/experiment/4_t1_t2_experiments.ipynb): Measure coherence properties through standard relaxation and dephasing experiments.

## Advanced experiment workflows

- [Randomized benchmarking](../../examples/experiment/5_randomized_benchmarking.ipynb): Estimate gate performance through Clifford-based randomized sequences.
- [State tomography](../../examples/experiment/6_state_tomography.ipynb): Reconstruct quantum states from repeated measurements.
- [EF characterization](../../examples/experiment/7_ef_characterization.ipynb): Work with the higher transmon levels beyond the computational basis.
- [State classification](../../examples/experiment/8_state_classification.ipynb): Train and inspect readout-state classifiers.
- [CR calibration](../../examples/experiment/9_cr_calibration.ipynb): Calibrate cross-resonance interactions for two-qubit control.
