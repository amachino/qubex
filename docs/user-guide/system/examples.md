# `system` example workflows

This page highlights notebooks for configuration loading, system-model
inspection, and pre-execution configuration checks.

## Recommended starting points

- [ConfigLoader Example](../../examples/system/config_loader.ipynb): Load configuration files directly and inspect the resulting `ExperimentSystem`.
- [QuEL-1 skew adjustment workflow](../../examples/system/quel1_skew_adjustment.md): Check the current skew, update `skew.yaml`, and re-run measurements for synchronized QuEL-1 boxes.
- [QuEL-3 Experiment Configure Check](../../examples/system/quel3_experiment_configure_check.ipynb): Validate that a QuEL-3 system definition can be configured before you move on to measurement or experiment execution.
